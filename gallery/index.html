Unplugged

For this three-page website challenge for a fictional camp, I was given PSD files to show how each page of the site should look when viewed from mobile, tablet, and desktop. To create this site, I started with the HTML5 boilerplate, and coded it starting with the header and footer before moving on to individual pages. I utilized mobile-first development practices and media queries to make this site responsive so that it would look beautiful regardless of the device viewed on. The "About" page also utilized iframes to embed a map of the area into the page.

Check out the site, click between the pages, and check out how it looks on multiple devices (note that external links won't go anywhere as it is a fictional company).

Rogue Pickings

The Rogue Pickings site, although it consists of only a single page, was actually constructed twice. This was a project used in my coursework twice. When first created, I was given a PSD file and told to create code for a static webpage using floats and Divs that matched the image as closely as possible.

After creating this single page and matching it to the original design, I was instructed to change the site structure to utilize the Flexbox property of CSS instead of floats. This second assignment based on the same project also required the code to be changed to a responsive site utilizing mobile-first development techniques and mobile queries. If you check out the code on GitHub, you can see how drastically different the code looked before and after the structure change.
